# Task 6: JWT Authentication Rejection Tests

## Test Scenarios

### 1. Missing Authorization Header

Request:
```bash
curl http://localhost:8080/api/auth/me
```

Expected Response:
```json
{
  "error": "missing authorization header"
}
```
Status: 401 Unauthorized

### 2. Invalid Authorization Format

Request:
```bash
curl -H "Authorization: invalid-format" http://localhost:8080/api/auth/me
```

Expected Response:
```json
{
  "error": "invalid authorization header format"
}
```
Status: 401 Unauthorized

### 3. Invalid Token

Request:
```bash
curl -H "Authorization: Bearer invalid-token" http://localhost:8080/api/auth/me
```

Expected Response:
```json
{
  "error": "invalid or expired token"
}
```
Status: 401 Unauthorized

### 4. Expired Token

Request:
```bash
curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." http://localhost:8080/api/auth/me
```

Expected Response:
```json
{
  "error": "invalid or expired token"
}
```
Status: 401 Unauthorized

## Validation Error Tests

### Register - Missing Username

Request:
```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","password":"password123"}'
```

Expected Response:
```json
{
  "error": "validation failed",
  "details": {
    "username": "this field is required"
  }
}
```

## Implementation Details

### Middleware Behavior
- Extracts token from `Authorization: Bearer <token>` header
- Returns 401 if header is missing or malformed
- Validates JWT signature and expiration
- Sets user context for downstream handlers

### Password Security
- Passwords hashed with bcrypt (cost 10)
- Plain text passwords never stored
- Constant-time comparison to prevent timing attacks

### JWT Configuration
- Algorithm: HS256
- Access token expiry: 15 minutes
- Refresh token expiry: 7 days
- Secret from JWT_SECRET environment variable
