## Dependencies Added

- github.com/golang-jwt/jwt/v5 v5.2.2
- golang.org/x/crypto v0.37.0 (bcrypt)
- github.com/go-playground/validator/v10 v10.26.0

## Endpoints Implemented

- POST /api/auth/register - Creates user, returns JWT token
- POST /api/auth/login - Validates credentials, returns JWT token
- POST /api/auth/refresh - Refreshes access token
- GET /api/auth/me - Returns current user (requires JWT)

## Authentication Middleware

Validates JWT tokens from Authorization header:
- Header format: `Authorization: Bearer <token>`
- Returns 401 for missing/invalid tokens
- Sets user context (userID, username, email) for handlers

## Validation Rules

- Username: 3-50 characters, alphanumeric + underscore only
- Email: Valid email format required
- Password: Minimum 8 characters

## Notes

- Auth routes are only registered when DATABASE_URL is set and connection succeeds
- Without database, only health check is available
- Passwords hashed with bcrypt (default cost)
- JWT secret read from JWT_SECRET environment variable
- Access tokens expire in 15 minutes
- Refresh tokens expire in 7 days

## Expected Behavior with Database

When DATABASE_URL is configured:
1. Register creates user and returns valid JWT
2. Login validates credentials and returns JWT
3. Refresh generates new access token
4. /me returns user details with valid token
5. Middleware rejects requests without valid token
